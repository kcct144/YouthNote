# 修复 WordSelector 组件的重复词问题

## 问题分析
当前组件的问题：
- 使用 `correctWords` 字符串数组来判断单词是否正确
- 当句子中有重复词时，`includes` 方法会匹配到所有相同的词，无法区分位置

## 解决方案
修改组件，支持两种 `correctWords` 格式：

1. **字符串数组格式**（现有格式）：
   ```vue
   :correctWords="['We', 'love']"
   ```
   适用于无重复词的句子

2. **数字数组格式**（新格式）：
   ```vue
   :correctWords="[0, 1]"
   ```
   直接指定正确单词的索引位置，适用于有重复词的句子

## 技术实现
1. **修改 props 类型**：
   - 将 `correctWords` 类型改为 `string[] | number[]`

2. **修改判断逻辑**：
   - 添加类型判断，根据 `correctWords` 的类型选择不同的判断方法
   - 字符串数组：使用 `includes` 方法
   - 数字数组：直接检查索引是否匹配

3. **保持向后兼容**：
   - 确保现有使用方式仍然有效
   - 新格式仅在需要时使用

## 预期效果
- 无重复词的句子：继续使用现有格式
- 有重复词的句子：使用新的索引数组格式
- 组件能够正确判断单词的正确性，不受重复词影响
